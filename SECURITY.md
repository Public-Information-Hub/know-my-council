# Security policy

KnowMyCouncil is early-stage. Security handling is best-effort, and response times are not guaranteed unless explicitly stated elsewhere.

## Reporting a security issue

If you believe you have found a security vulnerability, **do not open a public issue**.

Preferred reporting routes:

1. **GitHub Security Advisories** (recommended, if enabled for this repo)
- Go to the repository's "Security" tab and use "Report a vulnerability".

2. **Private email**
- Email: **[REPLACE_WITH_SECURITY_CONTACT_EMAIL]**
- Please include: affected component, version/commit, steps to reproduce, and impact assessment.

If you are unsure whether something is a security issue, report it privately anyway.

## What counts as a security issue

Examples include:

- authentication or authorisation bypass (current or future)
- remote code execution
- SQL injection, SSRF, path traversal
- sensitive data exposure (credentials, tokens, personal data)
- supply chain compromise

## What not to report publicly

- live credentials, API keys, or tokens
- personal data
- exploit code that can be used against running systems

## Disclosure

We aim to:

- acknowledge receipt
- reproduce the issue (when possible)
- patch and release a fix
- communicate impact and remediation steps

Coordinated disclosure timelines vary based on severity and maintainer availability.
