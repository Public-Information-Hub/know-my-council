# Governance (early-stage)

KnowMyCouncil is early-stage but aims to be structured from day one. This document describes how decisions are made and how the project stays evidence-led and non-partisan.

## Roles

### Project lead

The project lead is responsible for:

- setting direction and priorities
- maintaining project standards (quality, security, evidence principles)
- final decisions when consensus cannot be reached

### Maintainers

Maintainers are responsible for:

- reviewing and merging pull requests
- triaging issues
- maintaining release hygiene and security response processes

Maintainership is a responsibility, not a status.

See: [MAINTAINERS.md](MAINTAINERS.md).

## Decision making

We prefer:

1. Discussion in an issue or discussion thread
2. A clear proposal with trade-offs
3. Evidence and reproducibility for factual claims and data changes
4. Small reversible steps where possible

For technical decisions, we aim for rough consensus. If consensus cannot be reached, the project lead makes the final call with an explanation.

## Non-partisan and evidence-led expectations

KnowMyCouncil is focused on public-interest transparency.

- We do not use partisan framing.
- We separate evidence from interpretation.
- We document data provenance and retain sources where lawful and practical.

See: [docs/data-and-evidence-principles.md](docs/data-and-evidence-principles.md).

## Disagreements and disputes

If there is disagreement:

- assume good faith
- ask for sources and reproducible steps
- avoid personal attacks and speculation

If needed, maintainers may:

- request a neutral rephrase
- ask for additional evidence
- pause a change until discussion resolves

If a dispute involves conduct, follow [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Adding maintainers

New maintainers may be added when they have a track record of:

- constructive participation
- evidence-led work
- good judgement on trade-offs and security
- ability to follow through on review and maintenance commitments

The project lead proposes additions; existing maintainers provide input; the final decision is recorded in a GitHub issue.

## How this may evolve

If the project grows, governance may expand to include:

- a maintainer council
- formal editorial roles for evidence and investigation workflows
- community moderation processes

Any change to governance should be proposed and discussed publicly.
