# Recommended GitHub labels

GitHub labels are not version-controlled by default, so this file documents a suggested label taxonomy for this repository.

## General

- `bug`: Something is broken or incorrect in the software
- `documentation`: Documentation changes or issues
- `enhancement`: Improvement to existing behaviour
- `good first issue`: Suitable for new contributors
- `help wanted`: Maintainers would welcome external help
- `needs discussion`: Proposal or direction needs agreement before implementation

## Data and evidence

- `data issue`: Missing data, broken sources, suspected incorrect records
- `source needed`: A claim is missing primary source links
- `investigation`: A lead or pattern to research and validate

## Project management

- `infrastructure`: Local dev infra, dependencies, tooling
- `governance`: Governance, moderation, contributor policy
- `security`: Security-related work items (do not use for reporting vulnerabilities; see SECURITY.md)

## Dependencies

- `dependencies`: Automated dependency update PRs (e.g. Dependabot)
